python面向对象高级特性

发表于 2019-11-14 | 分类于 Python

最近在用python的时候遇到了一些之前没有遇到过的一些面向对象的一些方法，查了一些资料系统的了解了一下，其实并不复杂

1.属性

这里要介绍的属性其实和init函数里面定义的属性相似，适用于一些属性需要通过其他的一些属性经过复杂的计算而得出，这些属性我们通过一个方法来进行定义

这个方法就是普通的方法，但重点在于加了property的装饰器,在我们调用的时候直接 类名.pay 就可以了

2.staticmethod的classmethod

staticmethod 静态方法： 让类里的方法，直接被类调用，就像正常的函数一样

classmethod 类方法： 默认参数cls，可以直接用类名调用，与类属性交互

所以在类中方法有三种:

  • 普通方法: 由对象调用；至少一个self参数；执行普通方法时，自动将调用该方法的对象赋值给self；
  • 静态方法: 由类调用； 至少一个cls参数；执行类方法时，自动将调用该方法的类复制给cls
  • 类方法: 由类来调用，无默认参数

3.isinstance和issubclass

isinstance(obj,cls)检查是否obj是否是类 cls 的对象

issubclass(sub, super)检查sub类是否是 super 类的派生类

4.反射

反射的概念: 通过字符串的形式操作对象相关的属性

下列方法适用于类和对象（一切皆对象，类本身也是一个对象）

常用：hasattr、getattr

不常用：setattr、delattr

5. str 和 repr

我们都知道在python里面，list是一种数据类型，而且也是一个类。当我们以list()来创建一个序列时，我们也就创建了一个List对象。如上面的lst，知道了lst是List类型的对象后，我们print(lst)时，为什么没有直接打印出对象的内存地址，而是直接打印出给对象赋的值呢？其实，这一切是因为在list类的内部，实现了str功能

我们可以通过定制str方法来实现打印别的信息

str和repr必须return 字符串
当打印一个对象的时候，如果实现了str，打印str中的返回值，当str没有被实现的时候，就会调用repr方法（即str和repr同时存在时，会执行str，不会执行repr）
但是当你用字符串格式化的时候 %s和%r会分别去调用str和repr

不管是在字符串格式化的时候还是在打印对象的时候，repr方法都可以作为str方法的替补
但反之不行

7. new 和 del

new:创建对象时被调用，比init先执行

有以下要注意的几点:

  • new至少要有一个参数cls，代表要实例化的类，此参数在实例化时由Python解释器自动提供
  • new必须要有返回值，返回实例化出来的实例，这点在自己实现new时要特别注意，可以return父类new出来的实例，或者直接是object的new出来的实例
  • init有一个参数self，就是这个new返回的实例，init在new的基础上可以完成一些其它初始化的动作，init不需要返回值

del：在对象被删除或者程序执行完毕后，被调用

8. item系列

getitem_setitem___delitem

我们在列表中学过这种取元素的方式。比如说lst = [1,2,3,4],取第一个元素 lst[0],又或者在字典中dd ={‘name’:’xiaohua’} 取元素dd[‘name’]。实际上，这种取元素的方式，与getitem,setitem,delitem三个函数有关。
用法如下:

  • getitem：当我们想要按照 obj[attr]方式调用对象的属性时，触发这个函数的执行
  • setitem：当我以f[‘name’] = ‘xiaohua’ 方式修改对象属性值的时候，会触发该函数的执行。
  • delitem：当以del f[‘name’] 方式删除对象的属性值时，会触发這个函数的执行
