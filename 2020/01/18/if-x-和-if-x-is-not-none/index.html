if x 和 if x is not none

发表于 2020-01-18 | 分类于 Python

今天在使用刷算法题的时候，出现了一个问题，我在调用if num is None的时候发现了问题，从而发现了在python中if x 和 if x is not None是不等价的

在上面的代码中，报错提示我数组越界，唯一的可能就是if num is None这个判断没有奏效，于是我试了试一个空列表，会不会通过判断，结果和想象中的不同，果然没有成功

然后我换成了 if num，结果就提示成功了。

于是我发现if x 和 if x is not None
if not x 和 if x is None
以上的式子都不是等价的！！！

原因：

当使用 if x is None的时候，操作符是is，用来检查x的id。None在python里是单例，当使用if x的时候，不同类型是有一些不同的。
如果期望值是一个boolean类型，而此时假设x不是一个boolean类型，则此时python会自动地调用nonzero方法。例如，if x事实上是这样被执行的，if x.nonzero或者(bool(x))。 nonzero是一个被命名得很反人类的名字，在python3中叫做bool它用来检测一个类的boolean值。
bool(None)返回的是False，所以，如果x是None，那么if x返回的也是False

但是python里面的其他值也被检测为False。最常见的例子就是空列表，bool([])也返回False。但是，空列表有一个隐含的意思，它不等于None！！None意味着没有值，而空列表意味着零个值，这在语义上是不同的！

因为之前经常使用的if num，理所当然地认为每次调用if num就是判断是否为空。于是就出现了这种问题。
